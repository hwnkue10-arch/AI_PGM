# 3. 팩토리얼 (Factorial)

팩토리얼은 수학적으로 다음과 같이 정의된다.

## 정의

n! = n × (n-1) × ... × 1

## 특징

- 반복문으로 쉽게 구현 가능
- 누적 곱 계산 문제의 대표 예제

## 핵심 아이디어

- 1부터 n까지 곱을 누적한다
- 반복문으로 순차적으로 계산한다

---

# 4. 재귀 함수 (Recursion)

재귀 함수는 자기 자신을 다시 호출하는 함수이다.

---

## 4.1 구조

재귀 함수는 반드시 두 부분이 필요하다.

### 1) 종료 조건 (Base Case)
- 무한 호출 방지

### 2) 재귀 호출 (Recursive Case)
- 자기 자신을 호출

---

## 4.2 팩토리얼 (재귀)

팩토리얼은 재귀로 표현하기 좋은 대표 문제이다.

- n! = n × (n-1)!

---

## 4.3 피보나치 (Fibonacci)

피보나치 수열은 다음과 같다.

## 정의

- 0, 1, 1, 2, 3, 5, 8, ...

## 규칙

- F(n) = F(n-1) + F(n-2)

---

## 특징

- 재귀 구조 이해에 좋음
- 계산 비용이 높음 (비효율적일 수 있음)

---

# 5. 반복 vs 재귀 비교

| 반복문 | 재귀 함수 |
|--------|-----------|
| 속도가 빠름 | 상대적으로 느림 |
| 메모리 효율 좋음 | 스택 사용 |
| 구조 단순 | 수학적 표현에 적합 |

---

# 3. 리스트 (List)

리스트는 여러 개의 데이터를 하나의 변수에 저장할 수 있는 자료형이다.

## 특징

- 순서가 있다 (index 존재)
- 수정 가능 (mutable)
- 중복 허용
- 다양한 자료형 저장 가능

## 기본 사용

- 대괄호 `[]`로 생성
- 요소는 콤마(,)로 구분

---

## 리스트의 주요 기능

- 추가 (append)
- 삭제 (remove, del)
- 수정 (index 접근)
- 탐색 (in, index)

---

# 3.1 리스트 슬라이싱

리스트의 일부 구간을 잘라서 사용하는 기능이다.

## 기본 구조

- `list[start:end]`
- `list[start:end:step]`

## 특징

- 원본 리스트는 유지됨
- 부분 데이터 추출 가능

## 활용

- 일부 구간 추출
- 역순 출력
- 간격 선택

# 4. Set 자료형

Set은 중복을 허용하지 않는 자료형이다.

## 특징

- 중복 데이터 제거
- 순서가 없음 (index 없음)
- 빠른 탐색 속도

## 사용 목적

- 중복 제거
- 집합 연산 (합집합, 교집합, 차집합)

---

## Set의 핵심 특징

- 순서가 없다 → 인덱싱 불가능
- 중복 허용 안 함
- 수정 가능 (mutable)

---

# 5. 클래스 (Class)

클래스는 객체를 만들기 위한 설계도이다.

## 객체(Object)

- 실제로 생성된 인스턴스
- 클래스의 실체

---

## 5.1 클래스 구성 요소

### 1) 생성자 (__init__)

- 객체가 생성될 때 자동 실행
- 초기값 설정 역할

---

### 2) 속성 (Attribute)

- 객체가 가지는 데이터
- 예: 채널, 볼륨, 상태

---

### 3) 메서드 (Method)

- 객체가 수행하는 기능
- 클래스 내부 함수

---

## 5.2 __str__ 메서드

- 객체를 출력할 때 문자열 형태를 정의
- print() 결과를 제어

---

## 5.3 Getter / Setter

### Getter
- 값 조회 기능

### Setter
- 값 변경 기능

---

## 클래스 특징 요약

- 코드 재사용성 증가
- 현실 세계 모델링 가능
